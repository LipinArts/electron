WHAT IS ELECTRON?

  • Open source library for creating cross-platform desktop applications with HTML, CSS and JS
  • Combines Chromium and Node.js into single runtime
  • Apps can be packaged for Window, Mac and Linux

History of Electron

  • Project began in January 2013 and formally knows as Atom shell (originally was written for Atom)
  • Project was open sourced in May of 2014
  • Renamed to Electron in April of 2015
  • Released version 1.0.0 in June of 2016

It has since become a popular tool used by open source developers, startups and established companies

Electron under the hood

    Creates Browser Window instances to run web page.
  • Renderer Process - Each webpage runs its own isolated process called renderer prcess.

Electron structure

Package.json

{
  "name": "new_app",
  "version": "1.7.9",
  "main": "main.js",
  "scripts": {
    "start": "electron ."
    }
}

Main.js

const {app, BrowserWindow} = require('electron')
let mainWindow = null;
app.on('ready', () => {
  mainWindow = new BrowserWindow({ width: 500, height: 400 });
  mainWindow.loadURL('file://' + __dirname + '/index.html');
});

index.html

<html>
  <head>
    <script>
      window.onload = () => {
        const fs = require('fs');
        const p = document.createElement('p');
        p.textContent = fs.readFileSync('cats.txt');
        document.body.appendChild(p);
      }
    </script>
  </head>
  <body>
    <h1>Electron is awesome.</h1>
  </body>
</html>

Why Electron App

Electron app vs Web app

  • There is no browser dependency, which can mean avoiding costs in the long-term.
  • An Electron app will run exactly the same in 10 years as it does now. It won’t be affected by new or broken browsers and, from a user perspective, will look and behave the same.
  • You don’t need to worry about users having to manage multiple open browsers and tabs.
  • Users can view Electron applications on a variety of different browsers, including Chrome, Firefox, Internet Explorer, Edge and Safari.
  • Electron applications run the same regardless of the operating system.
  • You don’t need to worry about browser caches when the website is updated.
  • Users won’t lose their data when they refresh.
  • Development cost for Electron applications is lower because developers don’t need to code across multiple browsers and operating systems.
  • You dont have to worry about slow internt connection or lack of it at all. Electron app works offline.
